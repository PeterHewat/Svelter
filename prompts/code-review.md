- **Code Quality**: Adherence to best practices and coding standards
- **Architecture**: Component design, separation of concerns, and modularity
- **Performance**: Potential bottlenecks, unnecessary reactivity, or inefficient operations
- **Security**: Potential vulnerabilities or security anti-patterns
- **Type Safety**: TypeScript usage, proper typing, and error handling
- **Testing**: Testability and coverage considerations
- **Accessibility**: UI components following a11y best practices
- **Tech Stack Compliance**: Proper use of Svelte 5, SvelteKit, Convex, Clerk, Tailwind
- **Maintainability**: Code clarity, documentation, and future extensibility
- **Edge Cases**: Potential failure scenarios and error states

Provide specific suggestions for improvements with code examples where applicable.
